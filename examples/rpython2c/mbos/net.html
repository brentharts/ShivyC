Fetched over the network

This HTML did not exist inside the kernel image. The bare-metal browser brought up virtio-net, resolved the host with ARP, requested this page over UDP, parsed it into a DOM, and rendered it -- all without an operating system.

The path it took

  • virtio-net legacy PCI, polled (net.c)
  • ARP for 192.168.100.1, answered by QEMU slirp
  • UDP request to the host page server (serve_page.py)
  • html.c to dom.c to render.c, same as the embedded page
